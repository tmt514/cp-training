Combinatorial Objects

We can use recursion to generate a certain type of combinatorial objects, such as subsets, combinations, and permutations. They serve one very important problem solving skill — enumeration.

When designing a recursive function, one key concept is to realize the meaning of the depth of recursion. The following question can be used to check this:

Sanity Check

In recursion depth-ddd, do you focus on the ddd-th item from the input or the ddd-th item you've selected?

Subsets​

The simplest idea for enumerate subsets is to focus on each item ddd: choose it or not?

  • Java
  • Python
import java.util.LinkedList;

public class printsubsets {

  public void printAllSubsets(int n, int d, LinkedList<Integer> currentSubset) {
    if (d > n) {
      System.out.println(currentSubset);
      return;
    }
    currentSubset.add(d);
    printAllSubsets(n, d + 1, currentSubset);
    currentSubset.removeLast();
    printAllSubsets(n, d + 1, currentSubset);
  }

  public static void main(String[] args) {
    printsubsets solver = new printsubsets();
    solver.printAllSubsets(5, 1, new LinkedList<Integer>());
  }
}

Algorithm Design Questions​

  1. Can you reduce the number of parameters for printSubsets() function?
  2. What happens if we change currentSubset.removeLast(); to currentSubset.remove(d); on line 12?

Combinations​

If we decide at ddd-th depth which element is chosen as the ddd-th element, then we are able to list all combinations of a given set.

  • Java
  • Python
import java.util.LinkedList;

public class printcombinations {

  public void printAllCombinations(int n, int k, int d, int now, LinkedList<Integer> current) {
    if (d > k) {
      System.out.println(current);
      return;
    }
    while (now <= n) {
      current.add(now);
      printAllCombinations(n, k, d + 1, now + 1, current);
      current.removeLast();
      now++;
    }
  }
  public static void main(String[] args) {
    printcombinations solver = new printcombinations();
    solver.printAllCombinations(5, 3, 1, 1, new LinkedList<Integer>());
  }
}

Algorithm Design Questions​

  1. Why do we need the parameter now?
  2. Can you reduce the number of parameters for printSubsets() function?
  3. Can you modify the above implementation such that it prints out all subsets instead?

Permutations​

Permutations are very similar to combinations, except that k=nk=nk=n and we don't need the extra now parameter. However, we need to pay extra attention that the same element must not appear twice.

  • Java
  • Python
import java.util.HashSet;
import java.util.LinkedList;

public class printpermutations {
  
  public void printAllPermutations(int n, int d, HashSet<Integer> used, LinkedList<Integer> current) {
    if (d > n) {
      System.out.println(current);
      return;
    }
    for (int i = 1; i <= n; i++) {
      if (!used.contains(i)) {
        used.add(i);
        current.add(i);
        printAllPermutations(n, d + 1, used, current);
        current.removeLast();
        used.remove(i);
      }
    }
  }
  public static void main(String[] args) {
    printpermutations solver = new printpermutations();
    solver.printAllPermutations(4, 1, new HashSet<Integer>(), new LinkedList<Integer>());
  }
}

Algorithm Design Questions​

  1. Why do we need the parameter used?
  2. In a deeper depth of recursion, the program actually waste lots of time testing used.contains(i) without any progress. Is it possible to speed up?
