Exact Pattern Matching

The task is very simple. You are given a long text T[1..n]T[1..n]T[1..n] and a pattern P[1..m]P[1..m]P[1..m]. The goal is to find all occurrences of PPP inside TTT as a substring.

Naive Algorithm​

Knuth-Morris-Pratt (KMP) Algorithm​

The KMP Algorithm computes a prefix function prefix[i]prefix[i]prefix[i] of the text.

Z-Function​

ZZZ function is different from the prefix function. Z[i]Z[i]Z[i] is defined to be the longest common prefix of the given string and the ``suffix of that string starting from the iii-th character''.

Extensions​

High level intuition: usually there are some collection of texts T={T1,T2,…,Tk}\mathcal{T}=\{T_1, T_2, \ldots, T_k\}T={T1​,T2​,…,Tk​} and some patterns P={P1,P2,…,Pℓ}\mathcal{P}=\{P_1, P_2, \ldots, P_\ell\}P={P1​,P2​,…,Pℓ​}. The goal for the exact pattern matching algorithms would be to find/detect if a pattern occurs in the text. In this section we focus on the find all occurrences problem:

For each pattern PjP_jPj​, the goal is to find for all iii, the locations where PjP_jPj​ appears in TiT_iTi​.

Four Settings​

  • One text, one pattern.
  • Multiple texts, one pattern.
  • One text, multiple patterns.
  • Multiple texts, multiple patterns.
